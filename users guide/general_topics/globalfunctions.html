EasyDev Users Guide

Global Function

توابع عمومی سیستم بدون نیاز به بارگذاری کتابخانه یا فایل خاصی در دسترس میباشند.

 

 

تابع loadClass: این تابع جهت بارگذاری انواع کلاسها و کتابخانه ها به کار میرود، نحوه استفاده از این تابع به شکل زیر میباشد ( مقدار برگشتی تابع آدرس کلاس یا کتابخانه درخواست شده میباشد. ):

$var =& loadClass('file-name', 'class-name', 'object-name', 'dir');

'file-name': مقدار این پارامتر نام فایل کلاس یا کتابخانه درخواستی بدون پسوند php. میباشد. ( نوع: رشته )

'class-name': اسم کلاسی که میخواهید فراخوانی کنید. ( نوع: رشته )

'object-name': در صورتی که مقداری برای این پارامتر مشخص کنید، کلاس با همین اسم در سیستم در دسترس خواهد بود ( در صورتی که مقداری برای این پارامتر تعیین نکنید به صورت پیشفرض از اسم کلاس استفاده میشود ). در صورتی که مقدار این پارامتر برای یک کلاس که بیشتر از یکبار با دستور loadClass فراخوانی میشود متفاوت باشد، سیستم به صورت خودکار برای هر مقدار یک آدرس جداگانه تولید خواهد کرد که این به معنی است در حافظه برای هر مقدار فضاهای جداگانه ای تخصیص داده خواهد شد. در صورتی که در یک بخش دیگر احتیاج به مقادیر کلاسی که قبلا فراخوانی کرده اید دارید باید مقدار این پارامتر در هر دو بخش یکسان باشد.

'dir': در صورتی که برای این پارامتر مقداری مشخص نشود سیستم به صورت خودکار فولدرهای libraries را برای بارگذاری کلاس مورد نظر جستجو خواهد کرد. در صورتی که مقداری برای این پارامتر مشخص کنید سیستم در مسیر مشخص شده کلاس مورد نظر را چستجو خواهد کرد. به طور مثال اگر کلاس مورد نظرتان در مسیر app/myapps قرار دارد مقدار این پارامتر باید به صورت 'myapps' وارد شود.

 

تابع isLoadedClass: این تابع اسم کلاسهای بارگذاری شده را به صورت یک آرایه برمیگرداند. نحوه استفاده از این تابع به شکل زیر میباشد:

$loadedClasses = isLoadedClass();

 

تابع getConfigItem: این تابع آیتم درخواستی از فایل Config را در اختیار شما قرار میدهد. نحوه استفاده به شکل زیر میباشد:

$defaultC = getConfigItem('defaultController');

در مثال بالا مقدار آیتم defaultController در متغیر defaultC$ قرار میگیرد.

برای اطلاع از اسامی آیتم ها به فایل config/config.php مراجعه کنید.

 

تابع setConfigItem: با استفاده از این تابع میتوانید مقادیر آیتم های موجود در فایل config/config.php را تغییر دهید. نحوه استفاده از این تابع به صورت زیر میباشد:

setConfigItem('defaultController', 'test');

در مثال بالا در پارامتر اول نام آیتمی که میخواهیم مقدار آن را تغییر بدهیم را وارد میکنیم و در پارامتر دوم مقداری که میخواهیم را وارد میکنیم.

 

تابع getSiteUrl: این تابع آدرس Url وب سایت را برمیگرداند. به طور مثال:

echo getSiteUrl();
// http://example.com/index.php

echo getSiteUrl(array('first', 'second'));
// http://example.com/index.php/first/second

این تابع آدرس وب سایت را به همراه آدرس صفحه اصلی یا index.php تولید میکند. در صورتی که طبق مثال دوم یک آرایه از رشته ها را به این تابع ارسال کنید، این رشته ها را به ترتیب وارد آدرس تولید شده میکند ( این تابع زمانی به کار میرود که میخواهید برای صفحات خود لینک ایجاد کنید ) در آموزش های آینده در مورد نحوه ایجاد لینک ها توضیح داده خواهد شد.

 

تابع getBaseUrl: این تابع آدرس Url وب سایت را برمیگرداند. نحوه استفاده از این تابع به صورت زیر میباشد:

echo getBaseUrl();
// http://example.com/

echo getBaseUrl(array('first', 'second'));
// http://example.com/index.php/first/second

تنها تفاوتی که این تابع با تابع getSiteUrl دارد این است که در صورتی که هیچ پارامتری برای آن ارسال نشود آدرس وب سایت را بدون درج آدرس صفحه اصلی تولید میکند.

 

تابع makeUriString: این تابع آرایه ای از رشته ها را دریافت کرده و به صورت یک رشته uri بازمیگرداند.

echo makeUriString(array('one', 'two', 'three'));
// one/two/three

echo getBaseUrl(array('first', 'second'));
// http://example.com/index.php/first/second

در صورتی که switchToQueryString را از config/config.php فعال کرده باشید رشته هایی که این تابع تولید میکند به شکل زیر خواهد بود

echo makeUriString(array('c' => 'one', 'm' = > 'two', 'd' => 'three'));
// ?c=one&m=two&d=three

echo getBaseUrl(array('c' => 'one', 'm' = > 'two', 'd' => 'three'));
// http://example.com/index.php?c=one&m=two&d=three

 

تابع lang: این تابع جهت چندزبانه کردن وب سایت ها به کار میرود. نحوه استفاده از این تابع بعدا در بخش آموزش Language توضیح داده خواهد شد.

 

تابع stod: این تابع جهت جایگزین کردن فاصله ها با کاراکتر - (DASH) مورد استفاده قرار میگیرد.

echo stod('The quick brown fox jumps over the lazy dog')
// The-quick-brown-fox-jumps-over-the-lazy-dog

این تابع urlTitle های مطابق با استانداردهای موتورهای جستجو تولید میکند.

 

تابع error: این تابع جهت مدیریت خطاها به کار میرود. عملکرد این تابع به صورت Redirect میباشد که وقتی اجرای برنامه به این خط برسد کاربر را به صفحه Error ارسال خواهد کرد.

error('error title', 'error message');

جهت ویرایش ظاهر صفحه Error به این مسیر app/errors/errors.php مراجعه کنید.

تابع writelog: این تابع مفید جهت ثبت وقایع و خطایابی مورد استفاده قرار میگیرد، نحوه استفاده از این تابع به شکل زیر میباشد.

writeLog('log title', 'log message');

جهت تغییر مسیر ذخیره فایل های Log و همچنین برای فعال و غیر فعال کردن مکانیزم Log، فایل app/config/log.php ویرایش کنید. مسیر پیشفرض برای ذخیره فایلهای Log فولدر logs میباشد.
